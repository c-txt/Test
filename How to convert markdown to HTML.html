    Software to convert markdown to html and their issues

    markdown #html #ssg

    https://wethat.github.io/MarkdownToHtml/ This one requires PowerShell from Microsoft which is easy enough to install on Mac but then running the build also requires a Windows Virtual machine > So UNusable because of too much hassles

    md2blog > This is the most promising but creates files with a dark background https://jaredkrinke.github.io/md2blog/

    blag https://github.com/venthur/blag Wasn't able to install using pip (some issue on my mac)

    bashblog Requires server access

    bashwrite Requires server access

    blogC++ Tried experiments to compile but couldn't

    BBSG Easy to install and start with git and docs But dependencies need to be installed

    markdown-to-folder https://github.com/joakin/markdown-folder-to-html?tab=readme-ov-file It just works!

    md2blog > This is the most promising but creates files with a dark background https://jaredkrinke.github.io/md2blog/

    docmd needs node and npm doesnt work on my machine

    okidoki Works well locally but not easily deployable

    Gaiden Outputs html files, but no search

    Pagegen Deplyment will be a challenge

    SiteGen https://github.com/RC-Lee/SiteGen?tab=readme-ov-file It works like markdown-to-folder, but not as well (doesn't convert .md links to .html links automatically; also the index page layout is very basic and doesnt show any folder structure)

    daux.io is promising but is not

    VS COde to batch export? https://deepwiki.com/yzhang-gh/vscode-markdown/3.2-markdown-to-html-export (Alongwith the Cline Helper and File Tree Generation extensions)
