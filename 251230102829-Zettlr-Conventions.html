    Zettlr Conventions

    File naming

    Spaces by hyphens (-) Reason: for better readability, as well as ease of insertion, plus search engines interpret hyphens as spaces Example: Bangalore-Metro-Map.pdf

    Filenames need to be short, descriptive, in title case, and not only numbers like an id. But they can and ideally should end with the id. So tldr: id+descriptive seperated by hyphens Reason: So they can be searched in finder also and return usable results Example: The-Epochalypse-in-Year-2038-241203112712

    Attachments

    tldr: each and every attachment must have a the unique id of the associated container file in its filename like a regular md file.

    Container file: It would be desirable for each such attachment to have an associated md file in markdown that links to that attachment with descriptive content about the attachment. The filename should be the same file name as the attachment but obviously the extension would be markdown. So both the container file and the attachment can be searched for with the same id number and also will be listed sequentially in ordered lists. E.g: 20260123110759-Bangalore-city-map.md references the attachment: 20260123110759-Bangalore-city-map.pdf

    So first create the .md file, copy the Zid and then rename the attachment. The .md file will have a desctiptive tile, desciptive name, and descriptive content but must include the Zid to be referencable.

    Tags

    Tags if they have multiple words need to seperated by hyphens and use lowercase e.g: #metro-map

    Unique ids

    Unique ids follow this format: z#: 251230104002 Generated by: "z#: %y%M%D%h%m%s" Zettlr pattern detection: (\d{12}) (Any sequence of 12 or more digits will be recognized)

    Zettelkasten Folder

    Do not specify a path to the Zettelkasten folder as though it makes creating new files through links, it makes managing them difficult especially while deleting. You may end up creating a new file through a spelling mistake and then you have to end up deleting them.
